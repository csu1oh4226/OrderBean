# 관리자 인터페이스 PRD 업데이트 보고서

**작성일**: 2025-12-15  
**작성자**: OrderBean Team  
**버전**: 2.0

---

## 📋 요약

제공된 와이어프레임을 기반으로 관리자 인터페이스 PRD를 업데이트했습니다. 재고 관리 기능과 주문 접수 기능이 추가되었습니다.

---

## 🎯 주요 변경 사항

### 1. 새로운 기능 추가

#### 재고 현황 관리
- **기능**: 메뉴별 재고 수량 표시 및 조정
- **UI**: 메뉴별 카드 형태, +/- 버튼으로 수량 조정
- **우선순위**: High

#### 주문 접수 기능
- **기능**: 대기 중인 주문을 접수 처리
- **UI**: 주문 항목 우측에 "주문 접수" 버튼
- **우선순위**: High

### 2. 대시보드 통계 개선

#### 주문 통계 요약
- **기존**: 총 주문 수, 총 매출, 오늘 주문
- **변경**: 총 주문 / 주문 접수 / 제조 중 / 제조 완료
- **형식**: 한 줄로 간결하게 표시

### 3. 주문 현황 섹션 추가

#### 주문 목록 표시
- 날짜 및 시간 표시 (예: "7월 31일 13:00")
- 주문 메뉴 및 수량 표시
- 주문 금액 표시
- "주문 접수" 버튼

---

## 📊 기능 비교

### 기존 PRD vs 업데이트된 PRD

| 기능 | 기존 | 업데이트 |
|------|------|----------|
| 대시보드 통계 | 총 주문, 매출, 오늘 주문 | 총 주문 / 접수 / 제조 중 / 완료 |
| 재고 관리 | ❌ 없음 | ✅ 추가 (메뉴별 재고 표시 및 조정) |
| 주문 목록 | 전체 주문 목록 | 대기 중인 주문 목록 |
| 주문 접수 | ❌ 없음 | ✅ 추가 (주문 접수 버튼) |
| 주문 상태 변경 | 준비 시작, 준비 완료 | 접수 → 제조 중 → 제조 완료 |

## 🎨 UI/UX 변경 사항

### 레이아웃 구조

#### 기존
```
헤더
├── 대시보드 (통계 카드 3개)
├── 피크 시간대 차트
├── 주문 관리 (전체 주문 목록)
└── 메뉴 관리
```

#### 업데이트
```
헤더
├── 관리자 대시보드 (주문 통계 요약)
├── 재고 현황 (메뉴별 카드 + +/- 버튼)
└── 주문 현황 (대기 주문 목록 + 주문 접수 버튼)
```

### 주요 UI 요소

1. **재고 카드**
   - 메뉴명
   - 재고 수량 (예: "10개")
   - "+" 및 "-" 버튼 (작은 정사각형)

2. **주문 항목**
   - 날짜/시간 (예: "7월 31일 13:00")
   - 메뉴 및 수량 (예: "아메리카노(ICE) x 1")
   - 가격 (예: "4,000원")
   - "주문 접수" 버튼 (우측)

3. **통계 요약**
   - 한 줄 표시: "총 주문 1 / 주문 접수 1 / 제조 중 0 / 제조 완료 0"

## 📝 API 변경 사항

### 추가된 API

#### 재고 관리 API

1. **GET /api/admin/inventory**
   - 재고 현황 조회
   - 메뉴별 재고 수량 반환

2. **PATCH /api/admin/inventory/:menu_id**
   - 재고 수량 조정
   - increase/decrease 액션 지원

#### 주문 관리 API

1. **GET /api/admin/orders/pending**
   - 대기 중인 주문 목록 조회
   - 기존 GET /api/admin/orders와 분리

2. **POST /api/admin/orders/:id/receive**
   - 주문 접수 처리
   - 새로 추가된 엔드포인트

### 수정된 API

#### GET /api/admin/dashboard

**기존 응답**:
```json
{
  "stats": {
    "totalOrders": 156,
    "totalRevenue": 2340000,
    "todayOrders": 23
  },
  "peakHours": [...]
}
```

**업데이트된 응답**:
```json
{
  "totalOrders": 1,
  "receivedOrders": 1,
  "preparingOrders": 0,
  "completedOrders": 0
}
```

## 🗂️ 데이터 모델 변경

### 추가된 모델

#### Inventory (재고)

```typescript
interface Inventory {
  menu_id: string
  menu_name: string
  stock: number
  updated_at: string
}
```

### 수정된 모델

#### Order Status

**기존**:
```typescript
status: 'PENDING' | 'PREPARING' | 'READY' | 'COMPLETED' | 'CANCELLED'
```

**업데이트**:
```typescript
status: 'PENDING' | 'RECEIVED' | 'PREPARING' | 'COMPLETED' | 'CANCELLED'
```

- 'READY' → 'RECEIVED'로 변경
- 'RECEIVED' 상태 추가 (주문 접수)
